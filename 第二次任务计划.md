
# 第二次任务计划

> 基于导师反馈与前期实验结果，制定本轮任务计划。本轮聚焦于**雾天小样本目标检测**，采用更严谨的实验设计和方法探索。

---

## 一、当前问题总结

### 1.1 实验问题
| 问题 | 现象 | 推测原因 |
|------|------|----------|
| 21 epoch 实验 | 第2 epoch 指标即达最高，后续震荡 | 验证集太小（~40张），个别样本检出与否即可大幅改变 mAP |
| 44 epoch 实验 | 训练损失稳定下降，第25 epoch 最高，后续过拟合 | 模型在学习，但数据量不足以支撑长时间训练 |
| 类别过多 | 公交车、卡车等各自仅几十个实例 | 400 张图不适合细分多类别，分类极不稳定 |

### 1.2 数据划分隐患
- 若数据来源于视频/连续帧，相邻帧高度相似，随机划分会导致**数据泄漏**（相似帧同时出现在训练集和测试集）
- 同一清晰图生成的不同浓度雾图必须放在**同一个集合**中

---

## 二、本轮核心研究方向

### 2.1 三个核心研究问题
1. **小样本方法**：在仅有少量真实雾天标注数据时，如何获得可靠检测模型
2. **合成雾数据的价值**：人工合成雾是否有帮助，如何使用才能避免合成到真实的域偏差（Domain Gap）
3. **去雾 vs 不雾**：去雾后再检测，是否真的比直接在雾图上训练更好

### 2.2 辅助研究方向
4. **雾天小目标漏检**：后续可针对雾天场景下的小目标漏检进行改进研究

---

## 三、实验设计改进

### 3.1 数据划分 — 五折交叉验证
| 项目 | 旧方案 | 新方案 |
|------|--------|--------|
| 划分方式 | 单次 8:1:1 随机划分 | **五折交叉验证** |
| 验证集 | ~40 张，指标不稳定 | 5 次轮换，报告 mAP 均值与标准差 |
| 同源约束 | 无 | 同源图（如同一帧的不同雾浓度）必须归属同一折 |

**具体操作**：
- 将真实雾天数据按"来源"（同视频段/同场景）分成 5 份
- 每次使用 4 份训练/微调，1 份测试，重复 5 次
- 报告最终 mAP 的均值（mean）和标准差（std）

### 3.2 类别简化
- **第一阶段仅保留 2 个类别**：`car`（车）和 `person`（人）
- 公交车、卡车等类别暂时合并为 `car`，或直接剔除
- 待基线稳定后，再逐步增加细分类别

### 3.3 骨干网络选择
- 二选一：**YOLOv8n** 或 **YOLO11n**（nano 版本，兼顾效果与效率）
- 建议统一使用同一个骨干进行对比实验

### 3.4 合成雾使用策略
- 将合成雾作为**训练增强（Augmentation）**，不将其视为独立的最终数据集
- 保证**同源图在同一集合**：同一张清晰图生成的 3 种浓度雾图，必须全部进入训练/验证/测试的同一部分
- **暂不同时处理**雨、雪、夜间等多场景（先聚焦雾天）

## 四、实验方案路线图

### 实验 A：Baseline — 直接在雾图上训练
- 数据集：真实雾天数据（仅 car + person 两类）
- 模型：YOLOv8n / YOLO11n
- 验证：五折交叉验证
- 目标：获得稳定的 baseline mAP

### 实验 B：合成雾数据增强
- 在 Baseline 基础上，使用合成雾数据作为训练增强
- 对照组：
  - B1：仅真实雾数据训练
  - B2：真实 + 合成雾混合训练
  - B3：仅合成雾训练，真实雾测试（评估 Domain Gap）
- 对比各组的 mAP 差异，评估合成雾的实际帮助

### 实验 C：去雾预处理 + 检测
- 流水线：雾图 → 去雾算法（DCP / AOD-Net / 其他） → 检测
- 对照组：
  - C1：直接雾图 + YOLO（实验 A 的 baseline）
  - C2：去雾后 + 模型 A（清晰图训练的 YOLO）
  - C3：去雾后 + 模型 B（雾天图训练的 YOLO）
- 回答：去雾到底有没有用？

### 实验 D：小样本方法探索（重点）
- 在仅有少量标注数据的前提下，尝试以下方法：
  - **迁移学习**：使用大规模清晰数据集（COCO）预训练，微调到雾天数据
  - **数据增强**：合成雾增强（同实验 B）
  - **冻结骨干**：仅微调检测头（Head），减少过拟合
- 用五折交叉验证对比不同小样本比例（10%、25%、50%、100%）下的性能变化

## 七、训练配置规范

### 统一实验配置
```
模型：YOLOv8n 或 YOLO11n（nanos）
图像尺寸：640×640
优化器：SGD / AdamW（保持一致）
Batch Size：根据 GPU 显存调整，建议 ≥16
Epochs：根据数据量调整，注意早停（Early Stopping）
验证策略：五折交叉验证
类别：car, person（仅两类）
```

### 训练注意事项
- 监控训练损失曲线，发现过拟合（训练损失持续下降但验证指标停滞/下降）时提前中止
- 避免 epoch 数过少导致的欠拟合，也避免过多导致的严重过拟合
- 记录每次实验的超参数和随机种子，确保可复现

## 八、预期产出

1. 五折交叉验证下的稳定 baseline（mAP 均值 ± 标准差）
2. 合成雾增强的消融实验结果（有 vs 无）
3. 去雾预处理的对比实验结果（去雾 vs 不雾）
4. 小样本设置下的方法比较（不同数据比例的性能曲线）
5. 代码规范、可复现的训练与评估脚本

---

> **最终目标**：不仅获得雾天检测性能的提升，更重要的是通过此项目完整体验视觉任务的**方法设计 → 实验验证 → 分析调优**的全流程，积累可迁移的研究能力。
