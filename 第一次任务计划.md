

基于 YOLOv11 进行恶劣天气（如雾天）下的目标检测（AWOD）是一个非常经典且具有极高应用价值的计算机视觉研究方向。

为了帮助您快速起步，我已经为您在工作区搭建了基础的代码目录，并创建了 requirements.txt。下面是启动和推进这个项目的**5步建议计划**：

### 1. 确定去雾算法与 YOLOv11 的结合策略
在正式写代码前，需要明确你们想怎么融合这两种算法。通常有以下三种流派（建议从 A 开始，逐渐向 C 演进）：
*   **策略 A：流水线/前处理 (Pipeline / Pre-processing)** 
    *   **原理**：图像 -> 传统去雾（如暗通道 DCP）或 深度学习去雾（如 AOD-Net / FFA-Net） -> 去雾后的增强图像 -> YOLOv11检测。
    *   **优点**：实现简单，两个模型完全解耦。
    *   **缺点**：推理速度慢；去雾模型的输出不一定是 YOLO 最喜欢的特征图，可能会丢掉一些对目标检测有用的边缘特征。
*   **策略 B：数据增强与域自适应 (Data-driven / Domain Adaptation)**
    *   **原理**：使用 Albumentations 等工具对清晰图像进行人工加雾模拟，然后直接交由原生的 YOLOv11 训练，让网络自己去学习如何抵抗雾天干扰。
    *   **优点**：几乎不需要改 YOLO 的网络结构，推理速度没有损失。
*   **策略 C：端到端特征级融合 (End-to-End Feature Fusion)**
    *   **原理**：修改 YOLOv11 的网络结构。比如将一个轻量级的去雾模块（ISP模块或 Image Restoration 网络）缝合在 YOLOv11 的主干网络（Backbone）前面或中间，联合一个 Loss（去雾Loss + 检测Loss）一起做训练。
    *   **优点**：性能上限高，模型体积和推理速度可以控制得比较好，是当前顶会发论文的主流切入点。

### 3. 环境配置
YOLOv11 的开发目前主要是基于 ultralytics 框架。您需要在终端运行以下命令来搭建带 GPU 加速的深度学习运行环境：

```bash
# 1. 建议先用 Anaconda 或 venv 创建独立的虚拟环境
conda create -n awod python=3.10
conda activate awod

# 2. 安装项目的基本依赖库 (Ultralytics 即 YOLO11 官方库)
pip install -r requirements.txt
```

### 4. 数据集准备 (AWOD 必备)
要验证去雾对检测的提升，不能只用普通的 COCO 数据集。您需要准备恶劣天气特制数据集，推荐下载以下几种之一：
1.  **Foggy Cityscapes**: 最经典的合成雾天数据集（适用于街景/自动驾驶）。
2.  **RESIDE 数据集中的 RTTS**: Reside 专门用来做带目标的雾天街景数据集，标注格式比较全。
3.  **DAWN 数据集**: 包含大量真实恶劣天气（雾、雨、雪、沙尘）。

### 5. 第一周的行动建议 (Milestone 1)
1. **建立 Baseline（基线）**：不要急着加去雾算法。先直接下载 RTTS 或 Foggy Cityscapes 数据，跑通原版 YOLOv11 的训练（`yolo train data=dataset.yaml model=yolo11n.pt`），记录下原始的 mAP（平均精度）。
2. **测试前处理效果**：在 dehaze 下写一个简单的 OpenCV 暗通道去雾脚本，将测试集图片先去雾，再送到刚才训练好的 YOLOv11 模型里，看看 mAP 是上升了还是下降了。这能帮你们直观感受到去雾的影响。
